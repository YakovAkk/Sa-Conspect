# Architectural Style and Pattern

> [!note]
> This note is created for Obsidian Canvas and includes styling settings for good visual structure.

## Architectural Style

Architectural styles work at a high level and help organize the structure of the code. They define an application's layers and main modules and explain how these parts connect and work together.

Examples of architectural styles include Layered, Client-Server, and Monolithic architectures. Each style focuses on specific elements and relationships, with rules guiding how they should be used.

## Architectural Pattern

Architectural patterns are common solutions to recurring problems in software design, especially within architectural styles. They define set subsystems, explain their roles, and provide rules for how they should work together.

Examples include MVC (Model-View-Controller) and MVVM (Model-View-ViewModel). These patterns show basic ways to structure software systems.
